# アプリのUI/UX — 「現状PSPOサイトの情報を整理して見せる」設計
date: 2026-06-21
status: approved direction (brainstorming) → implementation
基盤: 顧客アプリ(app/area/live/me)・集客(index)分離済み。

## 0. 狙い
公式サイトに“ある”が分散している情報を、アプリで**タスク指向に整理**し、
利用者の問い〈何ができる？／いくら？／どこで・自分の店は？／どう始める？／空いてる？〉に即答する。

## 1. 一次情報（実地調査 pspo.jp 2026-06-21）
- **業態30+**：24hジム(Technogym)/パーソナル/ヨガ・スタジオ(月~10)/マシンピラティス/BioCircuit/シミュゴルフ/ボルダリング/セルフエステ(男女別)/酸素カプセル/24hサウナ個室/P·STRETCH/ホワイトニング/カフェ(フリードリンク)/24h自習室/WORKS/カラオケ・ピアノ/イベントスペース/会員バー(半額)/ランドリー/キッズパーク(0~小3)/古着/ギャラリー/食堂/SPA(温泉)/Tポイント。
- **料金**：月額¥4,980(税込¥5,478)。オプション**らくらくプラン¥1,650/月**(水・ウェア・タオル)。初期=**入会金¥7,700＋事務手数料¥3,300**＋月額(最大2ヶ月+20日)。キャンペーン(事務無料コード/ペア入会/紹介¥3,000off)。初期費用シミュレータあり(結果は埋もれ)。
- **入会**：完全Web入会=**松山・東温・砥部のみ**(pspo24.hacomono.jp)。他は**店舗予約+書類**。必要=写真付きID＋クレカ/口座。未成年=保護者同伴・同意。
- **見学・体験**：有人**¥500/2h**・無人**無料2h**(7-19時・当日予約)。持ち物=室内履き/ウェア/タオル/水。
- **ルール**：**予約制**(無予約は待ち/2週間前~3時間前)・QR入館・マシンを戻す・消毒・会話は静かに・小中学生はカフェのみ。
- **店舗フィールド**：名称/住所(地図)/営業(24h・年中無休)/スタッフ時間/駐車(台数・無料)/電話/サービスtag/有人・無人/詳細・予約リンク。

## 2. アプリでの整理（4本柱）
### A. サービス・カタログ（js/pspo-info.js ＋ services.js 強化）
6グループ〈トレーニング/整う・ウェルネス/学ぶ・働く/食・くつろぐ/ビューティー/こども・その他〉。
各サービス：**何が・どこで(店舗数)・予約要否・条件(例:BEAUTYは登録要)**。タップ詳細を強化。
### B. 料金まるわかり（新規 price.html）
月額＋オプション＋初期＋キャンペーンの**明瞭な表** ＋ **初期費用シミュレータ**(開始日・プラン・らくらく有無・キャンペーンで総額計算)。
### C. 店舗詳細（area.html にボトムシート）
その店：営業/スタッフ時間・駐車・電話・**ここで使えるサービス**・有人/無人・予約要否・経路。
データが無い項目は正直に「詳細は公式店舗ページ」。
### D. はじめ方ガイド（新規 guide.html）
見学体験→入会(地域別フロー)→必要書類→未成年→持ち物→ルール/予約制 を順に。**予約制を埋もれさせない**。

## 3. 構造・ナビ
- 新規：`price.html`(料金)・`guide.html`(はじめ方)・`js/pspo-info.js`(整理済みデータ)。
- 強化：`area.html`(店舗詳細シート)・`js/services.js`(カタログ詳細)・`app.html`(導線カード)・`index.html`(料金/はじめ方リンク)。
- ナビ：app グループに 料金・はじめ方 を追加（タブバーは home/空き/マイ 維持）。app ホームに「サービス/料金/はじめ方/店舗」導線カード。

## 4. UX原則（PC/スマホ両対応・徹底）
タスク指向の見出し／Hick・Miller で塊化／予約要否・条件は**バッジで一目**／初期費用は数字で即答／
clamp・ブレークポイントで360–1440px／タップ56px・親指ゾーン／reduced-motion尊重／非PII・カメラレス維持。

## 5. 受け入れ基準
1. サービスがグループ＋〈何が/店舗数/予約要否/条件〉で表示される。
2. price.html で初期費用が**開始日・プラン・らくらく・キャンペーン**から総額計算される。
3. area で店舗タップ→詳細（営業/駐車/サービス/有人無人/予約/経路）。
4. guide.html で 見学体験→入会(地域別)→書類→未成年→持ち物→ルール が順に整理。
5. 360–1440px で崩れず、console error 0、非PII/カメラレス維持。

## 6. 検証
構文＋import解決＋jsdom(価格計算・サービスカタログ・店舗詳細描画)＋数値健全性(初期費用)＋serve＋敵対的レビュー(IA/UX/モバイル/正確性/コピー)。
