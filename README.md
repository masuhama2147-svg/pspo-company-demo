> 数値は一次分析・公開資料に基づく概算で、最新の公式情報とは差異があり得ます。検索インデックスは無効化（noindex）しています。

## クイックスタート

依存ゼロ。Node があれば動きます（ビルド不要）。

```bash
node serve.js
# → http://localhost:8731  （使用中なら自動で +1 して空きを探す＝ポートが被らない）
```

Python でも可：

```bash
python3 -m http.server 8731
```

> スマホ実機で見るときは同一 Wi-Fi で `http://<PCのIP>:8731`。
> ブラウザの「ホーム画面に追加」で**アプリ（PWA）**になります（＝現行の「アプリ→Chrome遷移」問題への回答）。

## 主な機能

**顧客アプリ**
- 🗺️ **空き状況マップ**（一目でわかる）— SVG模式図。`js/config.js` に **Google Maps API キー**を入れると実地図に切替（下記）。
- 📊 **店舗ライブ** — 占有ゲージ／ゾーン別ヒートマップ／設備の「あと何台空き」＋推定待ち時間／24h混雑予報／オフピーク誘導。
- 🙂 **パーソナライズ** — 通い方（ペルソナ）で画面が変わる。顔認証データの“還元”＝ランク／連続記録／**利用連動割引**／目標。
- 📣 **イベント/お知らせ**（右→左に流れる広告枠）、🔎 **店舗検索／エリア絞り込み**、📱 **PWA**（ホーム画面に追加・下部タブバー）。
- 🔒 **プライバシー** — 表示は人数・占有率だけ（顔・個票を持たない非PII設計）。

**経営AIエージェント（提案）**
- PSPO を「松山のデータゲートウェイ」と位置づける堀（MOAT）／AI×OSS エージェント群／信頼性・ガバナンス／
  civic・B2B データビジネス／競合ベンチ／**誠実な限界と正統性**。

## Google マップ連携（任意）

実地図にするには `js/config.js` の `GOOGLE_MAPS.apiKey` にキーを入れるだけ：

```js
export const GOOGLE_MAPS = {
  apiKey: "AIza...your-key...",          // Google Cloud → Maps JavaScript API を有効化して取得
  center: { lat: 33.84, lng: 132.77 },   // 松山周辺
  zoom: 9,
};
```

- キーが**空**なら SVG 模式図にフォールバック（そのまま動きます）。
- キーは**コミットしない**でください（Notion原則「秘密はコードに置かない」）。`.gitignore` 推奨。

## 画像について

## 技術（Notion プレイブック準拠）

- **ビルド無しの静的サイト**：素の HTML / CSS / JS＋ES Modules。「書いた JS＝動く JS」。
- **5層 CSS**：`base / layout / components / pages / motion`（CSS変数で統一・`c-` 接頭辞）。
- **単一の真実の源**：閾値・色・更新間隔・ブランドを `js/config.js` に集約。
- **イベント駆動**：状態が変わった時だけ描画。タブ非表示で停止＝無駄な実行を抑制。
- **非PII射影**：データは人数・占有率のみ（`js/data.js` は顔・個人を持たない）。
- **アクセシビリティ**：スキップリンク／タップ44px／入力16px／aria／`prefers-reduced-motion` 尊重。
- **PWA**：`manifest.webmanifest` ＋ `sw.js`（network-first：直したら即反映）。
- **依存ゼロのサーバ**：`serve.js`（ポート8731・使用中なら自動回避・HTML/CSS/JS は no-cache）。
